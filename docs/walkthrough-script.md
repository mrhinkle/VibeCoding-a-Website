## Recording Tips

### Screen Resolution & Size
- Use **1920×1080 or higher resolution** for clarity
- Ensure text in code/chat is readable at normal viewing distance
- Use **120% or 125% system font scaling** so text is large enough to read comfortably but doesn't crowd the screen
- Avoid dark mode on GitHub/Claude if possible (light backgrounds show better on camera)

### Microphone Setup
- Use a quality USB microphone (Blue Yeti, Rode NT1, or equivalent) positioned 6–12 inches from your mouth
- Record in a quiet room with soft furnishings (curtains, carpet, upholstered chairs absorb echo)
- Avoid recording near fans, AC units, or other background noise
- Test audio levels before recording — aim for -12dB to -6dB peak on your recording software

### Browser & Chat Interface
- Use **Chrome or Edge** for the clearest screen recording
- Maximize browser window so Claude chat is full-screen
- Use a simple light theme for Claude (default is fine)
- Keep browser tabs to a minimum to avoid visual clutter
- Disable notifications during recording

### Pacing
- Type prompts at **a natural, human pace** — not too fast, not too slow (60–80 words per minute feels right)
- Pause 2–3 seconds after typing a prompt before the agent responds (creates tension and lets viewer anticipate)
- When showing the preview or live site, let it load naturally — don't rush through
- Pause 1–2 seconds between major section transitions to let the viewer absorb what they saw

### Voice & Tone
- Speak as if explaining to a friend — conversational, not like reading a script
- Emphasis on **"you could do this too"** — make it feel achievable
- Avoid technical jargon; explain things in plain English
- Sound genuinely excited about what's being built

- **Total runtime goal:** 10 minutes exactly (account for intro fade-in and closing fade-out)
